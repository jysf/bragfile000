# SPEC-001: Go module + Cobra scaffold

## Context

First spec in STAGE-001 (foundations). Nothing else in PROJ-001 can
build until the repo has a working Go module, a compilable
`cmd/brag/main.go`, and the root `cobra.Command` that subsequent
subcommand specs will plug into. This spec stops just short of any
subcommands or any storage — those arrive in SPEC-002 (storage) and
SPEC-003/SPEC-004 (`add`, `list`).

Parent stage: `STAGE-001-foundations.md`. Project: PROJ-001 (MVP).

## Goal

Make `go build ./cmd/brag` produce a working binary whose `--version`,
`--help`, and `--db` flags behave exactly as `docs/api-contract.md`
specifies, with config-path resolution (`--db` → `BRAGFILE_DB` →
default) isolated in its own package and unit-tested.

## Failing Tests

Written now, during design. Build cycle's only job is to make these
pass (plus any internal helpers the implementer needs).

### `internal/config/config_test.go`

Imports: `testing`, `os`, `path/filepath`, `strings`, package under test.

- **`TestResolveDBPath_Default`** — `t.Setenv("BRAGFILE_DB", "")`,
  call `ResolveDBPath("")`. Assert: returned path ends with
  `/.bragfile/db.sqlite`, is absolute (`filepath.IsAbs`), and no
  `~` remains.
- **`TestResolveDBPath_FlagWins`** — `t.Setenv("BRAGFILE_DB", "")`,
  call `ResolveDBPath("/tmp/foo.db")`. Assert result equals
  `/tmp/foo.db`.
- **`TestResolveDBPath_EnvWhenFlagEmpty`** — `t.Setenv("BRAGFILE_DB",
  "/tmp/env.db")`, call `ResolveDBPath("")`. Assert result equals
  `/tmp/env.db`.
- **`TestResolveDBPath_FlagBeatsEnv`** — `t.Setenv("BRAGFILE_DB",
  "/tmp/env.db")`, call `ResolveDBPath("/tmp/flag.db")`. Assert
  result equals `/tmp/flag.db`.
- **`TestResolveDBPath_ExpandsTilde`** — `t.Setenv("BRAGFILE_DB", "")`,
  call `ResolveDBPath("~/weird.db")`. Assert no `~` in result, path
  is absolute, and it ends with `/weird.db`.

### `internal/cli/root_test.go`

Imports: `testing`, `bytes`, `strings`, `github.com/spf13/cobra`,
package under test.

All root_test cases use the **separate-buffers** pattern:
`var outBuf, errBuf bytes.Buffer; cmd.SetOut(&outBuf); cmd.SetErr(&errBuf)`.
This is the prescribed pattern for every subcommand test that follows
SPEC-001 — sharing one buffer hides violations of the
`stdout-is-for-data-stderr-is-for-humans` constraint.

- **`TestRootCmd_VersionFlag`** — Build `cmd := NewRootCmd("test-v0")`,
  `cmd.SetArgs([]string{"--version"})`, wire `outBuf`/`errBuf` per the
  pattern above, call `cmd.Execute()`. Assert: returns nil error,
  `outBuf` contains `"test-v0"`, `errBuf.Len() == 0` (nothing on
  stderr — this is the empirical check of the stdout/stderr split).
- **`TestRootCmd_HelpFlag`** — Same setup with args `["--help"]`.
  Assert: returns nil error, `outBuf` contains substrings `"--db"`
  and `"Usage:"`, `errBuf.Len() == 0`.
- **`TestRootCmd_NoArgs`** — args `[]`. Assert: returns nil error,
  `outBuf` contains `"Usage:"` (cobra's default when no subcommands),
  `errBuf.Len() == 0`.

Notes for the implementer on testing patterns:
- Always wire `cmd.SetOut(&outBuf)` and `cmd.SetErr(&errBuf)` to
  **separate** buffers. Asserting on a merged buffer would mask a
  stdout/stderr violation — the whole point of these tests is to
  enforce the split at the CLI boundary.
- Use `t.Setenv` (Go 1.17+) — it auto-restores env on test cleanup.
  Never `os.Setenv` directly in tests.

## Implementation Context

*Read before starting build. Self-contained handoff to the build

### Decisions that apply

- `DEC-003` — Config resolution order is flag → env → default. The
  `config.ResolveDBPath` implementation is the canonical embodiment
  of this decision. Keep it in `internal/config/config.go` so future
  code has one import for path resolution.
- `DEC-006` — Cobra is our CLI framework. `internal/cli/root.go`
  should use the conventional cobra patterns (see `cobra.dev` examples
  and any existing Go CLI like `gh`); don't invent your own abstractions
  on top.

### Constraints that apply

Paths this spec touches (`cmd/**`, `internal/cli/**`,
`internal/config/**`, `go.mod`, `go.sum`):

- `no-cgo` — SPEC-001 must not introduce a CGO-requiring dep. Cobra
  and pflag are pure Go; nothing else is added.
- `no-new-top-level-deps-without-decision` — Satisfied: cobra is
  justified by DEC-006. `pflag` rides along as cobra's internal dep
  (not a separate user-visible choice), so it does not need its own
  DEC.
- `errors-wrap-with-context` — `ResolveDBPath` wraps every error it
  returns with a `fmt.Errorf("resolve db path: %w", err)`-style
  message. `main.go` may just print to stderr and exit nonzero;
  wrapping isn't needed at the process boundary.
- `stdout-is-for-data-stderr-is-for-humans` — `--version` output is
  stdout (scriptable). Help text via cobra also goes to stdout by
  default. Error messages from `Execute` failures go to stderr.
- `test-before-implementation` — All tests in the Failing Tests
  section are written before any code. Run `go test ./...` — they
  fail (because the files don't exist yet). Then implement to turn
  them green.
- `one-spec-per-pr` — Branch is `feat/spec-001-go-module-and-cobra-
  scaffold`. Single PR.

- **Any subcommand** — `add`, `list`, `show`, `edit`, `delete`,
  `search`, `export`, `summary`. SPEC-003, SPEC-004, and beyond.
- **`internal/storage` package** — Created in SPEC-002. Do NOT add a
  storage dep (`modernc.org/sqlite`) in this spec. `go.mod` at the
  end of SPEC-001 lists only cobra and its transitive deps.
- **Migration runner / embedded SQL** — SPEC-002.
- **Subcommand registration helper (e.g., "add all subcommands")** —

- **Shell completion** — Cobra supports it for free, but the command
  that enables it (`brag completion`) is post-MVP polish.
- **Build-time version injection via ldflags** — Hardcode
  `Version = "dev"` as a package-level constant in `cmd/brag/main.go`

  STAGE-004's goreleaser setup.
- **CI config** — GitHub Actions workflow lands in STAGE-004.
- **`--db` actually opening a DB** — The root command defines the
  flag; it does not use it. Flag wiring only.

## Notes for the Implementer

- **Version handling.** Simplest shape: in `cmd/brag/main.go` declare
  `const Version = "dev"`, pass it into `cli.NewRootCmd(Version)`.
  Cobra's built-in `Version` field on `*cobra.Command` handles
  `--version` automatically when set.
- **Persistent flag.** `root.PersistentFlags().String("db", "",
  "path to bragfile db (default ~/.bragfile/db.sqlite)")`. Leave the
  default empty string — actual defaulting lives in
  `config.ResolveDBPath`. Future subcommands read the flag via
  `cmd.Flags().GetString("db")`.
- **Tilde expansion.** `os.UserHomeDir()` + `strings.HasPrefix(path,
  "~/")` is the standard pattern. Don't pull in a library for this.
- **`ResolveDBPath` return.** Always returns an absolute path. Call
  `filepath.Abs` before returning so downstream callers don't have
  to repeat it.
- **No `init()` functions.** Build the root command inside a
  constructor so tests can call `NewRootCmd` repeatedly without
  process-global state.
- **Package name for `cmd/brag`** is `main`. Package name for
  `internal/cli` is `cli`. For `internal/config`, `config`.
- **Reminder:** `go mod init github.com/jysf/bragfile000`, then
  `go get github.com/spf13/cobra@latest`, then `go mod tidy`.
